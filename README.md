# Task Manager API

REST API для управления задачами, реализованное на **FastAPI**. Поддерживает создание и получение задач с автоматической валидацией входных данных и генерацией интерактивной документации.

> *Текстовые описания эндпоинтов, моделей данных и кодов ошибок сгенерированы и отредактированы с помощью ИИ в соответствии с требованиями задания. Логика и архитектура API реализованы вручную.*

---

## Быстрый старт

### Требования
- Python 3.10 или выше
- Windows 10/11 с поддержкой PowerShell

### Установка и запуск
1. Откройте PowerShell в папке проекта
2. Создайте и активируйте виртуальное окружение:
   ```powershell  
   python -m venv venv  
   .\venv\Scripts\activate.bat  
3. Установите зависимости:  
 ```powershell
   pip install fastapi "uvicorn[standard]"
```
4. Запустите сервер разработки:
 ```powershell 
   python -m uvicorn main:app --reload  
```
Сервер будет доступен по адресу: http://localhost:8000  

### Документация API
После запуска откройте интерактивную документацию:  
* Swagger UI: http://localhost:8000/docs  
* ReDoc: http://localhost:8000/redoc  
**Эндпоинты:**
|Метод|Путь|Описание|Успех|Ошибки|  
|--|--|--|--|--|
|POST//tasks|Создание новой задачи с валидацией входных данных|201 Created|422 Unprocessable Entity|  
|GET|/tasks/{task_id}|Получение полной информации о задаче по ID|200 OK|404 Not Found|  
  
**Модели данных (Pydantic)**  
TaskCreate (запрос)   
|Поле|Тип|Описание|  
|--|--|--|
|title|string|Обязательная строка. Ограничение: 3–100 символов. Назначение: краткая идентификация задачи.|
|description|string / null|Необязательное поле. Назначение: детализация требований или контекст задачи.|
|priority|integer|Обязательное число. Диапазон: 1–5. Назначение: определение порядка обработки.|  
TaskResponse (ответ)  
|Поле|Тип|Описание|
|--|--|--|
|id|integer|Уникальный системный идентификатор. Генерируется автоматически.|
|title|string|Заголовок задачи (возвращается без изменений).|
|description|string / null|Описание задачи или null.|
|priority|integer|Приоритет задачи (1–5).|
Коды ошибок  
|Код|Название|Причина|Как исправить|
|--|--|--|--|
|404|Not Found|Задача с указанным task_id не найдена или удалена|Проверьте корректность ID в URL. Убедитесь, что задача существует.|
|422|Unprocessable Entity|Ошибка валидации: отсутствуют обязательные поля, нарушена длина title или priority вне диапазона 1–5|Изучите тело ответа API, исправите указанные поля и повторите запрос.|

### Примеры запросов
Создание задачи
```bash
url.exe -X POST "http://localhost:8000/tasks" \
  -H "Content-Type: application/json" \
  -d "{\"title\": \"Настроить CI/CD\", \"description\": \"Подключить GitHub Actions\", \"priority\": 4}"  
```
#### Получение задачи по ID 
bash
curl.exe "http://localhost:8000/tasks/1"
  
#### Попытка получить несуществующую задачу
```bash
curl.exe "http://localhost:8000/tasks/999"
# Ответ: {"detail":"Task not found"} (код 404)
```

### Ограничения
Данные хранятся в оперативной памяти (dict). При перезапуске сервера все задачи удаляются.  
Для production-использования рекомендуется подключить базу данных (SQLAlchemy, SQLModel или Tortoise ORM) и реализовать пагинацию, фильтрацию и аутентификацию.  

### Лицензия
Учебный проект. Код распространяется свободно для образовательных целей.
